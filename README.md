## ğŸ§± System Architecture

**Key Components:**

* **User Interface (UI):** Upload, view, verify, and manage legal documents
* **Blockchain Layer:** Stores cryptographic hashes of documents
* **IPFS:** Decentralized storage for actual document files
* **MetaMask:** Secure wallet integration for blockchain transactions

**Workflow:**

1. User uploads a legal document
2. Document is encrypted and stored on IPFS
3. A cryptographic hash (SHA-256) is generated
4. Hash is stored on the blockchain
5. Verification is done by comparing hashes
